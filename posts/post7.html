침입 탐지 및 방지 시스템(IDS/IPS)

동작 원리부터 유형, 탐지 방법, 장단점, 그리고 IDS와 IPS의 차이점까지 정리하였습니다.

침입 탐지 및 방지 시스템

네트워크 보안에서 공격을 막으려면 먼저 무슨 일이 일어나고 있는지 볼 수 있어야 한다. 여기서 등장하는 게 IDS와 IPS다. 둘 다 침입이나 이상 행위를 찾아내는 보안 장비지만, 가장 큰 차이는 탐지만 하느냐, 아니면 탐지하고 바로 차단까지 하느냐에 있다.

IDS 의심스러운 행위를 탐지하고 관리자에게 알려주는 역할
IPS 탐지한 위협을 실시간으로 차단하는 역할

이 글에서는 IDS와 IPS를 단순히 용어로만 외우는 게 아니라, 실제 네트워크에서 어떤 위치에 배치되고 어떤 기준으로 공격을 판단하는지까지 같이 살펴본다. 특히 둘의 차이는 시험이나 면접에서도 자주 나오는 부분이라, 탐지 방식, 배치 위치, 장단점을 연결해서 이해하는 게 중요하다.

  • IDS와 IPS의 기본 개념과 역할 차이
  • Inline, Out-of-band, Mirror Port 같은 네트워크 배치 방식
  • Signature, Anomaly, Hybrid 기반 탐지 방식
  • 실제 운영 시 SIEM, 방화벽, EDR과 함께 쓰이는 구조
핵심 한 줄 요약 — IDS는 “탐지 후 보고”, IPS는 “탐지 즉시 차단”이라고 보면 된다. IPS는 IDS의 기능을 포함하면서 한 단계 더 나아간 능동적 방어 시스템이라고 생각하면 편하다.
01. Definition

IDS와 IPS란 무엇인가?

IDS는 Intrusion Detection System의 줄임말이고, 침입 탐지 시스템이라고 부른다. 네트워크나 시스템에서 의심스러운 활동이 발생하면 그걸 찾아내고 관리자에게 경보를 보내는 보안 솔루션이다.

쉽게 말하면 IDS는 CCTV에 가깝다. 누군가 수상한 행동을 하면 바로 화면에 잡고 경보를 울려준다. 하지만 직접 문을 잠그거나 침입자를 막아내지는 않는다. 판단하고 대응하는 건 보안 담당자의 몫이다.

IPS는 Intrusion Prevention System의 줄임말이고, 침입 방지 시스템이라고 부른다. IDS처럼 공격을 탐지하는 것에서 끝나지 않고, 탐지된 위협을 자동으로 차단하거나 연결을 끊는 기능까지 수행한다.

PASSIVE / IDS

IDS

네트워크 트래픽을 모니터링하면서 이상 행위를 탐지하고 경고를 발생시킨다. 트래픽 흐름에는 직접 끼어들지 않기 때문에, 일단 알려주고 사람의 판단을 기다리는 방식이다.

ACTIVE / IPS

IPS

탐지와 동시에 공격 트래픽을 자동으로 차단한다. 보통 트래픽이 지나가는 길목에 Inline으로 배치되어, 위험하다고 판단한 패킷을 실시간으로 막을 수 있다.

IDS는 “수상한 사람을 발견하고 알려주는 경비원”, IPS는 “문 앞에서 바로 막아서는 경비원”에 가깝다. 방화벽이 IP, Port, Protocol 같은 규칙을 기준으로 출입을 통제한다면, IDS/IPS는 트래픽 내용이나 행위 패턴을 보고 공격 징후를 찾아낸다는 점에서 차이가 있다.
02. Network Position

동작 원리 — 네트워크에서의 위치

IDS와 IPS는 둘 다 트래픽을 분석하지만, 네트워크에서 놓이는 위치가 다르다. 이 위치 차이 때문에 실제 역할도 달라진다. 아래 구조는 이해를 돕기 위한 예시이고, 실제 환경에서는 방화벽과 IPS의 위치가 보안 정책이나 장비 구성에 따라 달라질 수 있다.

IPS는 보통 Inline 방식으로 트래픽 경로 위에 직접 배치된다. 그래서 모든 패킷이 IPS를 통과하고, IPS는 위험하다고 판단한 패킷을 바로 버리거나 연결을 끊을 수 있다. 대신 장비 장애가 생기면 네트워크 흐름 자체가 영향을 받을 수 있다.

IDS는 Out-of-band 방식으로 동작하는 경우가 많다. 스위치의 Mirror Port 또는 SPAN Port를 통해 트래픽 복사본을 받아 분석하는 식이다. 네트워크 흐름에 직접 끼어들지 않기 때문에 안전하지만, 이미 지나간 트래픽을 보고 판단하는 구조라 즉시 차단은 어렵다.

포인트 — IPS는 길목에서 막는 장비고, IDS는 옆에서 지켜보는 장비다. 그래서 IPS는 빠른 차단이 가능하지만 장애 영향이 크고, IDS는 안전하게 감시할 수 있지만 차단은 사람이 따로 해야 한다.
03. Detection Methods

탐지 방법 3가지

IDS/IPS가 공격을 찾아내는 방식은 크게 세 가지로 볼 수 있다. 알려진 패턴을 찾는 방식, 평소와 다른 행동을 찾는 방식, 그리고 둘을 섞는 방식이다.

01 Signature-based Detection
02 Anomaly-based Detection
03 Hybrid Detection

시그니처 기반 탐지

Signature-based Detection은 이미 알려진 공격 패턴과 현재 트래픽을 비교하는 방식이다. 예를 들어 요청 URL에 ' OR '1'='1 같은 SQL Injection 패턴이 있거나, 특정 악성코드가 자주 사용하는 통신 문자열이 보이면 시그니처 DB와 매칭해서 공격으로 판단한다.

장점은 정확도가 높고 오탐이 상대적으로 적다는 점이다. 이미 알려진 공격에는 꽤 강하다. 대신 새로운 Zero-day 공격이나 기존 공격을 살짝 바꾼 변형 공격에는 약할 수 있다. 그래서 시그니처 DB 업데이트가 계속 필요하다.

이상 행위 기반 탐지

Anomaly-based Detection은 정상적인 네트워크 행동을 먼저 학습하고, 그 기준에서 많이 벗어난 행동을 이상으로 판단하는 방식이다. 예를 들어 평소에는 내부 서버가 특정 국가 IP와 통신하지 않았는데 갑자기 대량 통신을 하거나, 한 PC에서 짧은 시간 동안 여러 포트로 접속을 시도하면 이상 행위로 볼 수 있다.

이 방식은 알려지지 않은 신규 공격도 잡아낼 가능성이 있다. 하지만 정상 범위를 잘못 학습하면 문제가 생긴다. 평소와 조금만 달라도 공격으로 판단할 수 있어서 정상 트래픽을 공격으로 잘못 판단하는 오탐이 많이 발생할 수 있다.

하이브리드 탐지

Hybrid Detection은 시그니처 기반과 이상 행위 기반을 함께 사용하는 방식이다. 예를 들어 SQL Injection처럼 이미 알려진 패턴은 시그니처로 탐지하고, 평소와 다르게 트래픽이 급증하거나 비정상적인 접속 시도가 반복되는 경우는 이상 행위로 탐지하는 구조다.

요즘 IDS/IPS 솔루션은 대부분 한 가지 방식만 쓰기보다는 여러 방식을 섞어서 사용한다. 공격 방식이 계속 바뀌기 때문에, 단일 탐지 방식만으로는 한계가 있기 때문이다. 공격자는 탐지를 피하려고 요청을 인코딩하거나 패킷을 쪼개거나 정상 트래픽처럼 보이게 변형하기도 해서, 룰 업데이트와 지속적인 튜닝이 중요하다.

04. IDS Types

IDS의 유형

IDS는 어디를 감시하느냐에 따라 여러 유형으로 나눌 수 있다. 보통은 네트워크를 보는 NIDS, 호스트 내부를 보는 HIDS, 무선 환경을 보는 WIDS가 대표적이고, 추가로 특정 프로토콜이나 애플리케이션 계층을 집중 분석하는 PIDS/APIDS 형태도 있다.

NIDS

네트워크 기반

네트워크 세그먼트의 트래픽 전체를 감시한다. 스위치의 미러 포트에 연결해서 넓은 범위를 모니터링할 수 있고, 기업 환경에서 가장 흔하게 사용된다.

HIDS

호스트 기반

서버나 PC 같은 개별 호스트에 에이전트를 설치해서 시스템 로그, 파일 변경, 프로세스 실행 등을 감시한다. 네트워크 구간에서 암호화되어 보이지 않던 행위도 호스트 내부 로그나 프로세스를 통해 확인할 수 있다는 장점이 있다.

WIDS

무선 기반

Wi-Fi 환경을 집중적으로 감시한다. 비인가 AP, Rogue AP, 무선 스니핑, Deauthentication 공격처럼 무선 네트워크 특유의 위협을 탐지한다.

PIDS / APIDS

프로토콜 기반

특정 프로토콜이나 애플리케이션 계층을 집중 분석한다. HTTP/HTTPS 같은 프로토콜 규칙 위반이나 애플리케이션 단의 비정상 행위를 찾아낸다.

정리하면 NIDS는 “네트워크 전체 감시”, HIDS는 “개별 장비 내부 감시”, WIDS는 “무선 환경 감시”가 핵심이고, PIDS/APIDS는 “특정 프로토콜이나 앱 계층을 더 깊게 보는 형태”라고 보면 된다.
05. IDS vs IPS

IDS vs IPS — 핵심 차이점

IDS와 IPS는 같은 뿌리에서 출발했지만, 대응 방식과 배치 방식에서 차이가 확실하다. 가장 중요한 건 IDS는 관찰과 경보 중심이고, IPS는 차단과 대응 중심이라는 점이다.

구분 IDS IPS
대응 방식 탐지 후 경보를 보낸다 탐지 즉시 자동 차단한다
배치 위치 Mirror Port, Out-of-band Inline, 트래픽 경로 상
네트워크 영향 복사본 분석이라 네트워크 흐름에는 영향이 거의 없다 Latency가 발생할 수 있다
오탐 위험 경보만 발생해서 서비스 영향은 낮다 정상 트래픽을 차단할 수 있다
장애 시 영향 모니터링만 중단된다 네트워크 전체 흐름에 영향이 생길 수 있다
사람 개입 보안 담당자의 판단과 사후 대응이 필요하다 정책에 따라 자동 대응할 수 있다
적합 환경 컴플라이언스, 포렌식, 가시성 확보 실시간 방어, 고보안 환경
주의할 점 — IPS는 탐지한 내용을 바탕으로 트래픽을 바로 차단하기 때문에 오탐에 특히 주의해야 한다. 정상적인 업무 트래픽을 공격으로 잘못 판단하면 실제 서비스 장애로 이어질 수 있다. 그래서 처음부터 바로 차단 모드로 넣기보다는 Detection 모드로 로그를 보면서 정책을 튜닝하고, 안정화 후 Prevention 모드로 바꾸는 방식이 안전하다.
06. Pros & Cons

장점과 단점

IDS와 IPS는 서로 우열 관계라기보다는 목적이 다르다. IDS는 이상 행위를 찾아내고 알려주는 탐지 역할에 강하고, IPS는 위험하다고 판단한 트래픽을 바로 막는 차단 역할에 강하다. 대신 각각의 단점도 확실히 존재한다.

IDS

탐지 중심
IDS 장점
  • 네트워크 흐름에는 거의 영향을 주지 않는다
  • 다양한 트래픽 로그를 모아 포렌식 분석에 활용할 수 있다
  • 오탐이 발생해도 서비스가 바로 중단되지는 않는다
  • 복잡한 네트워크 환경에도 비교적 유연하게 배치할 수 있다
  • 컴플라이언스나 감사 목적에 잘 맞는다
IDS 단점
  • 실시간 공격 차단은 어렵다
  • 경보를 보고 사람이 직접 대응해야 한다
  • 경보가 너무 많아지면 Alert Fatigue가 생길 수 있다
  • 빠른 공격은 탐지 후 이미 피해가 발생했을 수 있다

IPS

차단 중심
IPS 장점
  • 실시간 자동 차단으로 즉각 대응할 수 있다
  • 24시간 무인 자동화 방어에 유리하다
  • 자동화 공격이나 빠른 공격에 효과적이다
  • 보안팀의 수동 대응 부담을 줄일 수 있다
IPS 단점
  • 오탐 시 정상 트래픽까지 차단할 수 있다
  • Inline 배치라 Latency가 발생할 수 있다
  • 장비 장애 시 네트워크 전체에 영향을 줄 수 있다
  • 초기 설정과 정책 튜닝에 시간이 필요하다
  • 암호화 트래픽은 복호화 설정이 없으면 내용 기반 분석에 한계가 있다
07. Use Cases

실제 활용 방법

실제 기업 환경에서는 IDS나 IPS 하나만 따로 두기보다는, 방화벽, SIEM, EDR 같은 다른 보안 솔루션과 함께 사용한다. 이걸 보통 Defense in Depth, 즉 다층 방어 전략이라고 부른다.

SIEM은 Security Information and Event Management의 줄임말이다. IDS/IPS, 방화벽, 서버, EDR에서 발생한 로그와 이벤트를 한곳에 모아서 분석하고, 서로 관련 있는 경보를 연결해주는 중앙 관제 시스템이라고 보면 된다.

주요 활용 시나리오

  • 기업 내부망으로 들어오는 공격 트래픽 탐지 및 차단
  • 클라우드 환경에서 의심스러운 통신 모니터링
  • 컴플라이언스 감사에 필요한 보안 로그 확보
  • 무선 네트워크에서 Rogue AP나 비인가 접속 탐지
  • 데이터센터 경계 구간에서 외부 공격 방어
  • OT/ICS 환경에서 비정상 프로토콜 사용 감시
권장 운영 방식 — IPS를 도입할 때는 처음부터 바로 차단 모드로 운영하지 않는 게 좋다. 먼저 IDS처럼 로그만 남기는 Detection 모드로 운영하면서 오탐을 줄이고, 정책이 안정화되면 Prevention 모드로 전환하는 방식이 훨씬 안전하다.

IDS/IPS에서 나온 경보는 SIEM과 연동해서 중앙에서 관리하는 게 좋다. 장비별로 로그를 따로 보면 전체 공격 흐름을 놓치기 쉽기 때문이다. 예를 들어 방화벽 로그, IDS 탐지 로그, EDR 이벤트가 SIEM에서 하나로 연결되면 공격자가 어떤 경로로 들어왔는지 훨씬 보기 쉬워진다.

IDS / IPS

탐지와 차단

네트워크 트래픽에서 공격 징후를 찾아내고, IPS의 경우 위험 트래픽을 바로 차단한다. 즉, 현장에서 공격을 발견하거나 막는 역할에 가깝다.

SIEM 중심

로그 통합과 분석

대표적으로 SIEM이 이 역할을 맡는다. 다만 SIEM만 단독으로 끝나는 건 아니고, 로그 관리 시스템, SOAR, XDR 같은 체계와 연결되어 경보 분석과 대응까지 이어질 수 있다.

Closing

마무리

처음에는 IDS와 IPS가 이름도 비슷하고 역할도 비슷해 보여서 헷갈렸다. 하지만 정리해보니 핵심은 생각보다 단순한 것 같다.
IDS는 이상 행위를 찾아내는 탐지 장비이고, IPS는 위험한 트래픽을 바로 막는 차단 장비다.

이번 내용을 정리하면서 흥미로웠던 점은 IDS와 IPS가 단순히 “탐지 장비”와 “차단 장비”로만 나뉘는 게 아니라, 네트워크에 어디에 배치되느냐에 따라 역할과 위험성이 크게 달라진다는 부분이었다. IDS는 안전하게 지켜보는 데 강하고, IPS는 직접 개입해서 막는 데 강하다는 차이가 꽤 인상적이었다.

중요한 건 둘 중 하나만 무조건 선택하는 게 아니라, 환경에 맞게 조합하는 것이다. IDS는 네트워크에서 어떤 일이 일어나는지 넓게 볼 수 있게 해주고, IPS는 공격이 실제 피해로 이어지기 전에 빠르게 막아준다. 여기에 SIEM을 연결하면 여러 장비의 로그를 한곳에서 분석할 수 있어서 공격 흐름을 훨씬 더 잘 파악할 수 있다.

결국 IDS/IPS를 이해한다는 건 단순히 보안 장비 이름을 외우는 게 아니라, 탐지 → 분석 → 차단 → 대응으로 이어지는 네트워크 보안의 흐름을 이해하는 것이다. 그래서 보안 공부를 할 때 IDS와 IPS는 꼭 한 번 제대로 정리해두면 좋은 개념이라고 생각한다.

[CTF] Hacktheon 예선 참가

문제를 해결하기 위해 어떤 관점으로 접근했고, 어떤 취약점 또는 핵심 로직을 이용했는지 정리한 라이트업입니다.

개요

Hacktheon 예선에서 총 3개의 문제를 해결하였고, 각 문제의 접근 과정과 핵심 포인트를 정리합니다.

문제 1

문제 이름 : simple sqli
분야     : Web

문제 설명

아이디와 비밀번호를 입력받는 칸이 있었고, 간단하게 SQL Injection을 이용해 푸는 문제였다.

접근 과정

문제 파일을 보고

admin' or '1'='1' --

admin' or '1=1'; #

등등을 시도하다가 role 이 admin 이 되어야 하는 것을 보고

admin' or role='admin' -- 을 시도하였는데 플래그 값이 출력되었다.

Flag

hacktheon2026{d0nt_f0rget_the_s1ng1e_qu0te}


문제 2

문제 이름 : Recover it!
분야     : pwn

문제 설명

문제에서 prob 파일을 제공하여 wsl 에서 해당 경로로 접근하여 푸는 문제였다.

접근 과정

풀이 과정

window 에서 wsl 을 이용하여 경로를 찾는 과정에서 좀 헤멨지만 IDA 로 prob 파일을 열어보니 아래 코드와 같이 입력값을 대조해서 Correct! 를 출력하는 것을 금방 알 수 있었다.

for (int i = 0; i < 64; i++) {
    input[i] ^= i + 0x67;
}

if (memcmp(input, cmptable, 64) == 0)
    puts("Correct");

따라서 입력값 → XOR 변형 → cmptable이랑 비교 하는 코드에서

정답 입력값은 아래와 같이 반대로 계산하면 된다.

입력값 = cmptable ^ (i + 0x67)

이걸 0번부터 63번까지 해서 전부 이어붙이면 Correct! 를 출력하는 것을 볼수 있다.

Flag

문제 3

문제 이름 : voice over
분야     : Ai

문제 설명

이 문제는 Target Sentence 를 보고 비슷한 목소리의 음성파일을 업로드하여 목소리와 텍스트가 일치하면 플래그가 출력되는 구조다.

접근 과정

풀이 과정

먼저 타켓 문장이 적혀진줄 모르고 문제에서 제공해준 샘플파일들을 넣어보았는데, text similarity 에서 낮은 숫자가 나와 플래그가 출력되지 않았다.

burp 로 request 조작이나 토큰이 중요한 줄 알고 여러 방법을 시도해보았지만 문제가 풀리지 않았다.

따라서 남은 방법은 진짜 목소리를 녹음하여 파일을 넣는 것이기 때문에 주어진 문장으로 음성 파일을 만들어주는 사이트를 찾아 제작하였더니 문제가 풀렸다.

Flag

hacktheon2026{b7d30e21e4106a6ca4d451a218f15a97}


마무리

세 문제를 풀면서 ai 에 많이 의지하며 문제를 풀었다. 이번이 핵테온에 참가한지 벌써 3회차라 많은 문제들을 풀고 싶었지만, 주어진 시간내에 정확한 방법으로 문제를 푸는게 쉽지많은 않았다. 포너블 문제도 풀어보고, 전보다는 미묘하게 늘었다고 느끼지만 아직 본선까지 도전해보려면 갈길이 한참 먼 것 같다. 따라서 앞으로 더 많은 문제들을 풀어보며 개념들을 정리하고, 분야별로 어떤 취약점들이 있는지 정확하게 정리하는 시간을 가져야 할 것 같다.

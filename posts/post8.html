OverTheWire Bandit 풀이 정리

WSL 설치부터 Bandit 접속, bandit0 → bandit25까지 진행하며 사용한 Linux 명령어와 문제 풀이 과정을 정리했습니다.

00. Environment

WSL 설치 및 Bandit 접속

WSL Ubuntu 환경에서 터미널을 열고 Bandit 서버에 SSH로 접속했다. Bandit은 기본 SSH 포트인 22번이 아니라 2220번 포트를 사용하므로 -p 2220 옵션을 붙였다. 처음 계정은 bandit0이고, 초기 password도 bandit0이다.

명령어설명
ssh bandit0@bandit.labs.overthewire.org -p 2220bandit0 계정으로 Bandit 서버에 접속한다. -p 2220은 SSH 접속 포트를 2220으로 지정하는 옵션이다.

03. Level 2 → 3

Level 2 → Level 3

파일명에 공백이 포함되어 있었다. 공백은 명령어 인자를 구분하는 기준이라서 파일명을 그대로 쓰면 여러 개의 인자로 나뉘어 인식된다. 그래서 파일명 전체를 따옴표로 감싸거나 공백 앞에 역슬래시를 붙여 하나의 파일명으로 처리했다.

cat "spaces in this filename"따옴표로 파일명 전체를 감싸 공백을 포함한 이름을 하나의 인자로 전달한다.
cat spaces\ in\ this\ filename공백 앞에 \를 붙여 escape 처리하는 방법이다.

문자열이 ROT13 방식으로 치환되어 있었다. ROT13은 알파벳을 13칸씩 밀어 바꾸는 방식이라서, tr 명령어로 소문자와 대문자 범위를 각각 13칸 이동한 문자 집합으로 치환했다.

Level 12 → Level 13

hexdump 형태의 파일을 원래 바이너리로 복원한 뒤, 여러 번 중첩된 압축을 풀어야 했다. 홈 디렉터리에서는 작업 권한이 불편할 수 있으므로 mktemp -d로 임시 디렉터리를 만들고, xxd -r로 hexdump를 되돌렸다. 이후 file로 파일 형식을 계속 확인하면서 gzip, bzip2, tar 압축을 순서대로 해제했다.

명령어설명

cp ~/data.txt .원본 파일을 작업 디렉터리로 복사한다.
xxd -r data.txt > ahexdump를 원래 바이너리 형태로 복원한다.
file a현재 파일 형식을 확인해 어떤 압축인지 판단한다.
gzip -d, bzip2 -d, tar -xf확인된 압축 형식에 맞게 압축을 해제한다.

Level 13 → Level 14

이번 레벨은 password가 아니라 sshkey.private 키 파일을 이용해 다음 계정에 접속하는 방식이었다. 처음에는 키 내용을 복사해 따로 저장해보려 했지만 형식이나 권한 문제로 실패했고, 이후 scp 명령어를 찾아 키 파일 자체를 로컬로 가져온 뒤 ssh -i 옵션으로 접속했다.

명령어설명
cat sshkey.privateprivate key 내용을 확인한다.
scp -P 2220 bandit13@bandit.labs.overthewire.org:~/sshkey.private ./Bandit 서버에 있는 키 파일을 현재 로컬 경로로 다운로드한다. scp는 포트 옵션이 대문자 -P이다.
chmod 600 sshkey.privateSSH private key 권한을 안전하게 설정한다.
ssh -i sshkey.private bandit14@bandit.labs.overthewire.org -p 2220지정한 private key를 사용해 bandit14 계정으로 접속한다.

31000~32000 범위 안에서 열려 있는 포트를 찾고, 그중 SSL로 통신하며 현재 password를 받는 포트를 찾아야 했다. nc -z로 포트 스캔을 한 뒤, 알맞은 SSL 포트에 접속하자 다음 레벨 접속에 사용할 RSA private key가 출력되었다.
